# Contributing to idle-less

Thank you for your interest in contributing to idle-less!

## How to contribute

### Bug reports

Found a bug? Please [open an issue](https://github.com/tvup/idle-less/issues/new?template=bug_report.yml) with:
- Steps to reproduce
- Expected vs actual behavior
- Your platform (OS, Docker version, architecture)

### Feature requests

Have an idea? [Open a feature request](https://github.com/tvup/idle-less/issues/new?template=feature_request.yml) describing your use case.

### Pull requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feat/my-feature`)
3. Make your changes
4. Test locally with `docker compose up -d`
5. Submit a pull request

### What we're looking for

- Bug fixes and reliability improvements
- Documentation improvements
- Install script enhancements
- Platform compatibility (new architectures, NAS devices, etc.)

## Development setup

```bash
git clone https://github.com/tvup/idle-less.git
cd idle-less
# Edit install.sh or scripts/*.sh
# Test with: bash install.sh --wakeforce
```
